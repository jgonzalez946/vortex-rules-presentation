El Problema

La lógica de negocio embebida en aplicaciones genera múltiples dolores de cabeza:

  • Ralentiza cambios: Cada ajuste requiere un ciclo completo de deploy
  • Falta trazabilidad: No se sabe quién cambió qué ni cuándo
  • Baja predictibilidad: Impacto en KPIs desconocido antes de producción
  • Escasa autonomía: Negocio depende 100% de IT para cambios simples

Governance Workflow

DRAFT → PENDING → APPROVED → PUBLISHED

✅ Aprobaciones M-of-N

Configurables por dominio. Workflow flexible según criticidad del cambio.

📋 Auditoría Completa

Quién, qué, cuándo, por qué. Historial inmutable de cambios.

🔄 Rollback Seguro

Versiónado semántico (major.minor). Snapshots estables para recuperación.

⚠️ Rejected: Si falla QA o no pasa aprobación, se rechaza con comentarios y vuelve a DRAFT.

Flujo de Trabajo: Demo

1️⃣

Autenticación

Login vía Keycloak OIDC/PKCE + selección de Tenant

2️⃣

Crear Regla

Authoring Wizard → Upload Excel/DMN/DRL → Validación → Guardar DRAFT

3️⃣

Simular y Testear

Sandbox con payload JSON → QA Simulator → Test Suite → Run

4️⃣

Aprobar y Publicar

Submit → Approval M-of-N → Publish → Hot-reload automático
